# 智慧社区服务系统 — Python + MySQL + FastAPI 后端开发计划

---

## 一、项目概述

### 1.1 项目背景

本项目为「智慧社区服务」小程序的后端服务，对应前端 uni-app (Vue 3) 小程序项目。前端包含 15 个页面，涵盖物业服务、社区公告、报修、投诉建议、缴费、积分、公共空间预约、快递、志愿活动、社区活动、帮助中心等完整功能模块。

**当前状态**：前端所有数据为硬编码 mock，未对接任何后端 API。`utils/request.js` 已预留 HTTP 请求工具，`BASE_URL` 为占位符 `https://api.example.com`。

### 1.2 技术栈

| 层级       | 技术选型              | 说明                               |
| ---------- | --------------------- | ---------------------------------- |
| 语言       | Python 3.11+          | 类型提示，async/await 支持         |
| Web 框架   | FastAPI 0.100+        | 高性能异步框架，自动 OpenAPI 文档  |
| ORM        | SQLAlchemy 2.0+       | 异步模式，声明式映射               |
| 数据库     | MySQL 8.0+            | InnoDB 引擎，utf8mb4 字符集        |
| 数据库迁移 | Alembic               | 版本化数据库变更管理               |
| 数据验证   | Pydantic V2           | 请求/响应模型校验                  |
| 认证       | python-jose + passlib | JWT Token + bcrypt 密码哈希        |
| 缓存       | Redis 7+              | Token 黑名单、验证码、热点数据缓存 |
| 文件存储   | 本地                 | 图片上传存储                       |
| 任务队列   | Celery + Redis (可选) | 异步任务（邮件、短信、统计）       |
|            |                       |                                    |

### 1.3 开发阶段规划

| 阶段                     | 周期        | 内容                                     |
| ------------------------ | ----------- | ---------------------------------------- |
| **P0 — 基础架构** | 第 1-2 周   | 项目骨架、数据库设计、认证体系、通用工具 |
| **P1 — 核心功能** | 第 3-5 周   | 用户、房产、公告、报修、投诉建议         |
| **P2 — 业务功能** | 第 6-8 周   | 缴费、积分、优惠券、公共空间预约         |
| **P3 — 拓展功能** | 第 9-10 周  | 快递、志愿活动、社区活动、帮助中心       |
| **P4 — 上线优化** | 第 11-12 周 | 性能优化、管理后台、部署上线             |

## 三、数据库设计

### 3.1 ER 关系总览

```
User ──1:N──→ Property         (用户拥有多套房产)
User ──1:N──→ RepairOrder      (用户提交多个报修)
User ──1:N──→ Feedback         (用户提交多条反馈)
User ──1:N──→ PointsRecord     (用户有多条积分记录)
User ──1:N──→ Coupon           (用户持有多张优惠券)
User ──1:N──→ Booking          (用户有多个预约)
User ──1:N──→ Package          (用户有多个快递)
User ──1:N──→ VolunteerRecord  (用户参加多个志愿项目)
User ──1:N──→ ActivityRegister (用户报名多个活动)
Property ──1:N──→ Bill         (房产有多笔账单)
Property ──1:N──→ RepairOrder  (房产有多个报修)
Space ──1:N──→ TimeSlot        (空间有多个时段)
Space ──1:N──→ Booking         (空间有多个预约)
VolunteerProject ──1:N──→ VolunteerRecord (项目有多个参与者)
Activity ──1:N──→ ActivityRegister       (活动有多个报名者)
```

### 3.2 数据表设计

#### 3.2.1 用户模块

```sql
-- 用户表
CREATE TABLE users (
    id              BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    uid             VARCHAR(20) NOT NULL UNIQUE COMMENT '用户唯一标识',
    name            VARCHAR(50) NOT NULL DEFAULT '' COMMENT '用户名',
    avatar          VARCHAR(500) DEFAULT '' COMMENT '头像URL',
    phone           VARCHAR(20) DEFAULT '' COMMENT '手机号(加密存储)',
    password_hash   VARCHAR(255) NOT NULL COMMENT '密码哈希',
    is_verified     TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否已实名认证 0否1是',
    bio_enabled     TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否开启生物识别',
    status          TINYINT NOT NULL DEFAULT 1 COMMENT '状态 1正常 0禁用',
    last_login_at   DATETIME DEFAULT NULL COMMENT '最后登录时间',
    created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at      DATETIME DEFAULT NULL COMMENT '软删除时间',
    INDEX idx_phone (phone),
    INDEX idx_uid (uid)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';

-- 用户标签表
CREATE TABLE user_tags (
    id          BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    user_id     BIGINT UNSIGNED NOT NULL COMMENT '用户ID',
    icon        VARCHAR(100) DEFAULT '' COMMENT '标签图标',
    text        VARCHAR(50) NOT NULL COMMENT '标签文字',
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户标签表';

-- 收货地址表
CREATE TABLE addresses (
    id          BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    user_id     BIGINT UNSIGNED NOT NULL COMMENT '用户ID',
    name        VARCHAR(50) NOT NULL COMMENT '收件人姓名',
    phone       VARCHAR(20) NOT NULL COMMENT '收件人电话',
    province    VARCHAR(50) DEFAULT '' COMMENT '省',
    city        VARCHAR(50) DEFAULT '' COMMENT '市',
    district    VARCHAR(50) DEFAULT '' COMMENT '区',
    detail      VARCHAR(200) NOT NULL COMMENT '详细地址',
    is_default  TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否默认地址',
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='收货地址表';
```

```sql
    is_active   TINYINT(1) NOT NULL DEFAULT 1,
```

## 四、API 接口设计

### 4.1 通用规范

**基础路径**：`/api/v1`

**认证方式**：Bearer Token (JWT)，请求头 `Authorization: Bearer <token>`

**通用响应格式**：

```json
{
    "code": 200,
    "message": "success",
    "data": {}
}
```

**分页响应格式**：

```json
{
    "code": 200,
    "message": "success",
    "data": {
        "items": [],
        "total": 100,
        "page": 1,
        "page_size": 10,
        "pages": 10
    }
}
```

**错误码规范**：

| 错误码 | 说明                   |
| ------ | ---------------------- |
| 200    | 成功                   |
| 400    | 请求参数错误           |
| 401    | 未认证/Token 过期      |
| 403    | 无权限                 |
| 404    | 资源不存在             |
| 409    | 资源冲突（如重复报名） |
| 422    | 数据验证失败           |
| 500    | 服务器内部错误         |

```
登录流程：
1. 用户提交手机号 + 密码 → 验证 → 返回 access_token + refresh_token
2. 前端每次请求携带 Authorization: Bearer <access_token>
3. Token 过期 → 前端调用 /auth/refresh → 返回新 access_token
4. 退出登录 → Token 加入 Redis 黑名单

Token 设计：
- access_token: 有效期 2 小时
- refresh_token: 有效期 7 天
- JWT Payload: { "sub": user_id, "exp": 过期时间, "type": "access|refresh" }
```

### 5.3 积分系统

```
积分获取规则：
- 缴费成功：消费金额 × 1 积分/元
- 志愿活动参与：10 积分/次
- 社区活动参与：5 积分/次

积分消耗规则：
- 公共空间预约：根据空间定价扣除（如 20 积分/小时）

```
